01

執行摘要

本系統以 AWS Serverless 為核心，建構一個從資料採集、ETL、AI 分析到 LINE 推播的全自動台股投資預警平台。系統採用 EventBridge → SQS → Lambda fan-out 架構，確保大量股票分析任務可水平擴展且具備容錯能力。

AI 層使用 Amazon Bedrock + Claude 4.5 Haiku，搭配 LLM Grounding 與 Output Validation 雙重防護，避免幻覺並確保推播內容品質。整體部署透過 Terraform + GitHub Actions 完全自動化，符合企業級 IaC 與 CI/CD 標準。

相較於 v1，v2 補上 SQS 解耦、DLQ 容錯、Secrets Manager、API Gateway Webhook 與 Schema 契約表，使整份規劃從「個人實作」進化為「企業級交付物」。

階段數量 5
AWS 服務數 12+
預估月成本 $8-15
每日處理股票 ~1000
分析延遲 < 15min

🆕 v2 版本相對於 v1 的關鍵升級

  • 補回 SQS + DLQ：實現 fan-out 並發處理與失敗訊息隔離
  • 升級 LLM 模型：Claude 3 Haiku → Claude 4.5 Haiku（最新可用版本）
  • 資料源策略：主源改用 TWSE OpenAPI，yfinance 降為備援
  • 新增 Secrets Manager：所有 API Key 走密鑰管理服務，杜絕環境變數明文
  • LINE Bot 補完 API Gateway Webhook 流程，加入 HMAC 簽章驗證
  • DynamoDB 加 GSI：支援跨股票查詢，Sparse Index 控制成本
  • 新增 Schema 契約表：階段間輸入/輸出 Schema 明確定義
  • 觸發時間調整：15:00 → 15:30（避開 TWSE 資料尚未齊備風險）
  • Lambda Provisioned Concurrency + ARM Graviton2：成本與體驗平衡

五階段工作細節

01

基礎設施與環境自動化 (IaC)

🎯 使用 Terraform 定義所有雲端資源，確保環境可重複部署、可版本控制
Provider 與區域配置

AWS 區域選 ap-northeast-1（東京）— 對台灣 ms 級延遲，Bedrock Claude 4.5 系列已全面上架。Backend State 存於 S3 + DynamoDB Lock。

IAM 最小權限設計 v2 強化

每個 Lambda 一個獨立 Role，例如 worker-lambda-role 僅有： s3:PutObject（指定 bucket）、dynamodb:PutItem（指定 table）、 sqs:DeleteMessage（指定 queue）、secretsmanager:GetSecretValue（指定 secret ARN）。

儲存層

S3：twstock-raw-{env}（原始）、twstock-curated-{env}（Parquet）； 開 Versioning、Lifecycle 90 天轉 Glacier。
DynamoDB：StockAnalysisTable + 兩個 GSI（詳見 Schema 章節），On-Demand 計費。

Secrets Manager v2 新增

建立 secret：line/channel-token、line/channel-secret、finmind/api-key。 所有 Lambda 啟動時透過 boto3 拉取並 @lru_cache 快取。

ECR 映像檔倉庫

建立 twstock-lambda repository，啟用 image scanning（CVE 自動掃描）、lifecycle 保留最近 10 個版本。

SQS Queue + DLQ v2 新增

stock-tasks-queue（Standard, visibility timeout 900s, batchSize 10）+ stock-tasks-dlq（最大接收次數 3）。 所有失敗訊息進 DLQ 等人工排查。

02

數據採集與 ETL 引擎 (Containerized Lambda)

🎯 建立穩定、具備容錯能力且資料源可切換的資料管線
Docker 化環境

基礎映像檔 public.ecr.aws/lambda/python:3.11-arm64（ARM Graviton2，成本省 20%）。 安裝 pandas、pyarrow、requests、boto3。

資料源 Strategy Pattern v2 重大修正

抽象 StockDataSource 介面，實作三種來源並建立 fallback chain：
① TWSEAPISource（主源，官方 OpenAPI 免費穩定）
② FinMindSource（備源，免費版 600 req/hr 足夠）
③ YFinanceSource（最終備援）
主源失敗自動切換，並寫入 CloudWatch Custom Metric 追蹤可用性。

抓取欄位

收盤價、開高低、成交量、三大法人買賣超、PE/PB；計算 MA5/MA20/MA60、RSI(14)、KD、MACD。

冪等性（Idempotency）

每筆 SQS 訊息含 StockID + TradeDate；Worker 寫入 DDB 時使用條件式寫入 attribute_not_exists(StockID)，重複觸發不會覆蓋。

持久化雙軌

Hot Path：DynamoDB 寫入即時特徵供 AI 分析使用（毫秒級讀取）。
Cold Path：S3 Parquet 按 year=/month=/day= partition 存放，未來 Athena 直接查詢。

03

AI 智能分析與決策 (LLM Integration)

🎯 引入 LLM 將技術指標轉化為人類可讀的投資建議，並嚴控幻覺
模型選擇：Claude 4.5 Haiku v2 升級

v1 規劃的 Claude 3 Haiku 已是舊型號。改用 claude-haiku-4-5-20251001： 推理品質提升、成本相當（詳見成本對比章節）。Bedrock ap-northeast-1 已上架。

Prompt Engineering

System Prompt 設定 AI 為「資深量化分析師」，明確規定： ① 僅使用輸入 JSON 中的數據； ② 禁止推測未提供的資訊； ③ 輸出 JSON 格式（含 signal、confidence、summary 三個欄位）。

Output Validation 層 v2 新增

Bedrock 回傳後用 Pydantic 驗證 schema； 若 confidence < 0.6 或 schema 不符 → 走 fallback 規則引擎（純技術指標訊號）； 確保 LINE 推播 100% 是結構化、可控的訊息。

穩定性參數

temperature=0、top_p=0.95、max_tokens=300。 確保同樣輸入產生同樣輸出，便於回測與品質追蹤。

Grounding 強化 v2 新增

System Prompt 內嵌「禁區清單」：禁止輸出明確買賣建議（合規）、禁止承諾收益、禁止提及輸入外的個股。 下游 Output Validator 也對這些禁區做關鍵字過濾雙重保險。

04

自動化部署與監控 (DevOps & Observability)

🎯 全自動部署，問題發生時主動告警，符合企業 SRE 標準
CI/CD Pipeline (GitHub Actions)

Push → Lint (ruff) → Test (pytest + moto mock) → Docker Build (multi-arch) → 推送 ECR → Terraform Plan → 人工 Approve → Terraform Apply → 更新 Lambda image。
使用 OIDC 而非長期 IAM Key，符合最佳實務。

觸發時間調整：15:00 → 15:30 v2 修正

TWSE 盤後資料 14:30 才齊備，原本 15:00 觸發有抓到舊資料風險。改 15:30 並加上資料新鮮度檢查（若 TradeDate ≠ today 拋告警）。

三層告警策略 v2 強化

Layer 1 INFO：DLQ 累積 ≥ 5 → SNS Email
Layer 2 WARN：資料源 fallback 觸發 → SNS Email + LINE
Layer 3 CRIT：Lambda 連續失敗 ≥ 3 次 → SNS SMS（緊急）

可觀測性（Observability）

所有 Lambda 開啟 X-Ray Tracing；CloudWatch Dashboard 顯示：每日成功率、p99 延遲、Bedrock token 使用量、月成本走勢。

05

前端輸出與使用者互動 (Interface)

🎯 雙向 LINE Bot：自動推播 + 隨時查詢
LINE 推播（Push API）

SNS → Lambda → LINE Messaging API；使用 Flex Message 排版，顯示股號、訊號類型、技術指標雷達圖、AI 摘要。

API Gateway Webhook 流程 v2 新增完整流程

使用者訊息 → LINE Platform 推送 Webhook → API Gateway (HTTPS) → Lambda； Lambda 內必須驗證 X-Line-Signature HMAC-SHA256 簽章，防偽造請求。

Query Lambda Provisioned Concurrency v2 新增

為避免 Docker Lambda cold start（可達 5-15 秒），查詢用 Lambda 開 Provisioned Concurrency = 1（月成本約 $3）。 使用者體驗從「等待約 10 秒」降到「< 500ms」。

查詢功能

支援指令：查詢 2330（個股最新分析）、訂閱 2330（加入推播清單）、取消 2330。 清單存 DynamoDB UserSubscriptions table。
